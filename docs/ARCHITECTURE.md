# Архитектура проекта obsidian-sync

Этот проект представляет собой монорепозиторий, содержащий два основных компонента: плагин для Obsidian (`plugin`) и бэкенд-сервер (`server`).

## Обзор

- **`plugin/`**: Клиентская часть, работающая внутри приложения Obsidian.
- **`server/`**: Бэкенд-сервис для хранения и синхронизации данных.
- **`ops/`**: Скрипты для обслуживания и администрирования сервера.

---

### 1. Плагин (`plugin/`)

Это плагин для Obsidian, написанный на TypeScript. Он отвечает за всю логику на стороне клиента.

- **Точка входа**: `plugin/main.ts` инициализирует плагин, регистрирует его компоненты и настройки.
- **Пользовательский интерфейс (UI)**:
    - `plugin/src/ui/sync-settings-tab.ts`: Создает вкладку настроек, где пользователь может управлять параметрами синхронизации.
    - `plugin/src/ui/status-controller.ts`: Управляет иконкой статуса синхронизации в строке состояния Obsidian.
- **Движок синхронизации (`sync`)**:
    - `plugin/src/sync/engine.ts`: Основная логика синхронизации. Он отслеживает изменения в файлах хранилища, взаимодействует с API сервера для отправки (`push`) и получения (`pull`) изменений.
    - `plugin/src/sync/crypto.ts`: Отвечает за шифрование и дешифрование файлов перед отправкой на сервер и после получения с него. Это обеспечивает сквозное шифрование (E2EE).
- **Настройки**: `plugin/src/settings.ts` определяет структуру настроек плагина и управляет их сохранением и загрузкой.

### 2. Сервер (`server/`)

Бэкенд-сервис, написанный на TypeScript с использованием фреймворка [Nitro](https://nitro.unjs.io/).

- **API Routes (`server/server/routes/api/`)**: Определяет RESTful API эндпоинты, которые использует плагин.
    - **Аутентификация и устройства**:
        - `device/register.post.ts`: Регистрация нового устройства (клиента).
        - `device/revoke.post.ts`: Отзыв доступа для устройства.
    - **Синхронизация**:
        - `sync/push.post.ts`: Принимает изменения от клиента.
        - `sync/pull.post.ts`: Отправляет клиенту изменения с других устройств.
        - `sync/state.get.ts`: Позволяет клиенту получить текущее состояние синхронизации.
    - **Хранилище блобов (CAS)**:
        - `blob/[hash].put.ts`: Загрузка файла (блоба) в хранилище.
        - `blob/[hash].get.ts`: Получение файла из хранилища по его хешу.
        - `blobs/missing.post.ts`: Позволяет клиенту проверить, какие блобы отсутствуют на сервере, для их последующей загрузки.
- **База данных (`db`)**:
    - Используется **Drizzle ORM** для взаимодействия с базой данных (конфигурация в `drizzle.config.ts`).
    - `server/server/db/schema.ts`: Определяет схему таблиц (устройства, операции синхронизации, файлы и т.д.).
    - Миграции хранятся в `server/drizzle/`.
- **Утилиты (`utils`)**:
    - `auth.ts`: Логика аутентификации и авторизации запросов.
    - `cas.ts`: Реализация логики **Content-Addressable Storage (CAS)**. Файлы хранятся на основе хеша их содержимого, что позволяет избежать дублирования данных.
    - `sync-core.ts`: Ключевая бизнес-логика обработки операций синхронизации на стороне сервера.

### 3. Операции (`ops/`)

Набор shell-скриптов для выполнения административных задач:
- `backup.sh`: Резервное копирование данных сервера.
- `gc.sh` (Garbage Collection): Очистка "мусора" — удаление из хранилища блобов, на которые больше нет ссылок в базе данных. Это вызывается через API эндпоинт `admin/gc.post.ts`.
- `restore-check.sh`: Проверка целостности резервной копии.

## Схема потока данных (Data Flow)

1.  Пользователь вносит изменения в своем хранилище Obsidian.
2.  Плагин (`engine.ts`) обнаруживает эти изменения.
3.  Содержимое измененных файлов шифруется (`crypto.ts`).
4.  Плагин отправляет зашифрованные данные и мета-информацию на сервер через `sync/push` и `blob/...` эндпоинты.
5.  Сервер проверяет подлинность запроса (`auth.ts`), обрабатывает операцию (`sync-core.ts`) и сохраняет данные в базу данных (Drizzle) и дисковое хранилище (CAS).
6.  На другом устройстве плагин периодически запрашивает изменения с сервера (`sync/pull`).
7.  Получив данные, плагин расшифровывает их и применяет изменения к локальным файлам в хранилище Obsidian.
